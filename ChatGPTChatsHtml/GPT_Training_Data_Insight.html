What MISRA rules say that identifiers must be unique?

ChatGPT


The MISRA (Motor Industry Software Reliability Association) guidelines are a set of best practice directives for programming in C (and there's also a set for C++). These guidelines are designed to ensure the safety, reliability, and maintainability of embedded systems. Among the MISRA rules, there are rules related to the uniqueness of identifiers to avoid ambiguity and potential issues in the codebase.
 
Here's a brief overview of the MISRA C:2012 rules related to identifier uniqueness:
 
Rule 5.1 (Required): External identifiers shall be distinct.
This means that no two global or file-scope identifiers should have the same spelling in a program.
 
Rule 5.2 (Required): Identifiers declared in the same scope and name space shall be distinct.
Within a given scope, identifiers shouldn't overlap.
 
Rule 5.3 (Required): An identifier declared in an inner scope shall not hide an identifier declared in an outer scope.
If you declare an identifier inside a function, for example, it shouldn't have the same name as an identifier declared outside the function in a higher scope.
 
Rule 5.4 (Required): Macro identifiers shall be distinct.
This prevents confusion or unexpected behavior due to macro replacement.
 
Rule 5.5 (Advisory): Identifiers shall be distinct from macro names.
This suggests that regular variable/function names shouldn't overlap with macro names to avoid confusion.
 
Rule 5.6 (Required): A typedef name shall be a unique identifier.
Each typedef should be distinct to ensure clarity.
 
Rule 5.7 (Advisory): An object or function identifier with external linkage shall not be reused.
This aims to avoid unexpected behavior when linking different code modules together.
 
Rule 5.9 (Advisory): An identifier shall not be reused.
This is a more general rule, advising against reusing identifiers across different name spaces.
 
It's essential to note that these rules, especially the advisory ones, don't mean you can't technically use non-unique identifiers. However, following the guidelines can prevent many bugs, confusions, and maintenance challenges that could arise from non-distinct identifiers.
 
For more detailed information on these rules, along with the justifications and examples, you might want to consult the MISRA C:2012 guidelines directly.

Are the MISRA C:2012 guidelines available for free?

ChatGPT


No, the MISRA C:2012 guidelines are not available for free. MISRA C:2012, like the other MISRA guidelines, is copyrighted material and is typically purchased as a document from MISRA or licensed distributors.
 
Organizations and developers who intend to follow the MISRA guidelines in their projects usually purchase the official document. It provides not only the rules but also the rationale behind each rule and examples of compliant and non-compliant code. This information is crucial for understanding the intent behind each guideline and ensuring correct application in real-world scenarios.
 
If you or your organization need to use the MISRA guidelines, it's essential to obtain a legitimate copy to ensure you have accurate and up-to-date information, and to support the continued development and maintenance of the guidelines.
